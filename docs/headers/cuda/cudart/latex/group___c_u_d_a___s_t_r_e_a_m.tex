Add a callback to a compute stream. 

\begin{DoxyNote}{Note}
This function is slated for eventual deprecation and removal. If you do not require the callback to execute in case of a device error, consider using \doxylink{group___c_u_d_a___e_x_e_c_ga79b25e2139e85793740fffc48ef82359}{cu\+Launch\+Host\+Func}. Additionally, this function is not supported with cu\+Stream\+Begin\+Capture and \doxylink{group___c_u_d_a___s_t_r_e_a_m_ga3cf7bb74b5cb1240199d203d850a0d0b}{cu\+Stream\+End\+Capture}, unlike \doxylink{group___c_u_d_a___e_x_e_c_ga79b25e2139e85793740fffc48ef82359}{cu\+Launch\+Host\+Func}.
\end{DoxyNote}
Adds a callback to be called on the host after all currently enqueued items in the stream have completed. For each cu\+Stream\+Add\+Callback call, the callback will be executed exactly once. The callback will block later work in the stream until it is finished.

The callback may be passed \doxylink{group___c_u_d_a___t_y_p_e_s_gga0cdead942fd5028d157641eef6bdeeaaaa0eed720f8a87cd1c5fd1c453bc7a03d}{CUDA\+\_\+\+SUCCESS} or an error code. In the event of a device error, all subsequently executed callbacks will receive an appropriate \doxylink{group___c_u_d_a___t_y_p_e_s_gaf18c3affd9d913e337e3794abeade307}{CUresult}.

Callbacks must not make any CUDA API calls. Attempting to use a CUDA API will result in \doxylink{group___c_u_d_a___t_y_p_e_s_gga0cdead942fd5028d157641eef6bdeeaaae23d0197c490ec332a43e55b167968a3}{CUDA\+\_\+\+ERROR\+\_\+\+NOT\+\_\+\+PERMITTED}. Callbacks must not perform any synchronization that may depend on outstanding device work or other callbacks that are not mandated to run earlier. Callbacks without a mandated order (in independent streams) execute in undefined order and may be serialized.

For the purposes of Unified Memory, callback execution makes a number of guarantees\+: 
\begin{DoxyItemize}
\item The callback stream is considered idle for the duration of the callback. Thus, for example, a callback may always use memory attached to the callback stream. 
\item The start of execution of a callback has the same effect as synchronizing an event recorded in the same stream immediately prior to the callback. It thus synchronizes streams which have been "{}joined"{} prior to the callback. 
\item Adding device work to any stream does not have the effect of making the stream active until all preceding host functions and stream callbacks have executed. Thus, for example, a callback might use global attached memory even if work has been added to another stream, if the work has been ordered behind the callback with an event. 
\item Completion of a callback does not cause a stream to become active except as described above. The callback stream will remain idle if no device work follows the callback, and will remain idle across consecutive callbacks without device work in between. Thus, for example, stream synchronization can be done by signaling from a callback at the end of the stream. 
\end{DoxyItemize}

Attach memory to a stream asynchronously. 

Enqueues an operation in {\ttfamily h\+Stream} to specify stream association of {\ttfamily length} bytes of memory starting from {\ttfamily dptr}. This function is a stream-\/ordered operation, meaning that it is dependent on, and will only take effect when, previous work in stream has completed. Any previous association is automatically replaced.

{\ttfamily dptr} must point to one of the following types of memories\+:
\begin{DoxyItemize}
\item managed memory declared using the {\bfseries{managed}} keyword or allocated with \doxylink{group___c_u_d_a___m_e_m_gab3c8db1c575451d53313b05b1c6db7f8}{cu\+Mem\+Alloc\+Managed}.
\item a valid host-\/accessible region of system-\/allocated pageable memory. This type of memory may only be specified if the device associated with the stream reports a non-\/zero value for the device attribute \doxylink{group___c_u_d_a___t_y_p_e_s_gga3b9f561d2a42733dde99b2cedcaa413aa5fdcdbe1dfc3ad5ec428c279e0efb9cd}{CU\+\_\+\+DEVICE\+\_\+\+ATTRIBUTE\+\_\+\+PAGEABLE\+\_\+\+MEMORY\+\_\+\+ACCESS}.
\end{DoxyItemize}

For managed allocations, {\ttfamily length} must be either zero or the entire allocation\textquotesingle{}s size. Both indicate that the entire allocation\textquotesingle{}s stream association is being changed. Currently, it is not possible to change stream association for a portion of a managed allocation.

For pageable host allocations, {\ttfamily length} must be non-\/zero.

The stream association is specified using {\ttfamily flags} which must be one of \doxylink{group___c_u_d_a___t_y_p_e_s_ga5fed3d210af1e9b08b64dbe317a51073}{CUmem\+Attach\+\_\+flags}. If the \doxylink{group___c_u_d_a___t_y_p_e_s_ggae58977b6b2ea67a3fb0e11949302308ea0b42aae6a29b41b734d4c0dea6c33313}{CU\+\_\+\+MEM\+\_\+\+ATTACH\+\_\+\+GLOBAL} flag is specified, the memory can be accessed by any stream on any device. If the \doxylink{group___c_u_d_a___t_y_p_e_s_ggae58977b6b2ea67a3fb0e11949302308ea8b59c62cab9c7a762338e5fae92e2e9c}{CU\+\_\+\+MEM\+\_\+\+ATTACH\+\_\+\+HOST} flag is specified, the program makes a guarantee that it won\textquotesingle{}t access the memory on the device from any stream on a device that has a zero value for the device attribute \doxylink{group___c_u_d_a___t_y_p_e_s_gga3b9f561d2a42733dde99b2cedcaa413aa33110e44c9cb6ead02f03ff6f6fd495e}{CU\+\_\+\+DEVICE\+\_\+\+ATTRIBUTE\+\_\+\+CONCURRENT\+\_\+\+MANAGED\+\_\+\+ACCESS}. If the \doxylink{group___c_u_d_a___t_y_p_e_s_ggae58977b6b2ea67a3fb0e11949302308ea4b70b6a5e039f61eccc6b8db3dfac442}{CU\+\_\+\+MEM\+\_\+\+ATTACH\+\_\+\+SINGLE} flag is specified and {\ttfamily h\+Stream} is associated with a device that has a zero value for the device attribute \doxylink{group___c_u_d_a___t_y_p_e_s_gga3b9f561d2a42733dde99b2cedcaa413aa33110e44c9cb6ead02f03ff6f6fd495e}{CU\+\_\+\+DEVICE\+\_\+\+ATTRIBUTE\+\_\+\+CONCURRENT\+\_\+\+MANAGED\+\_\+\+ACCESS}, the program makes a guarantee that it will only access the memory on the device from {\ttfamily h\+Stream}. It is illegal to attach singly to the NULL stream, because the NULL stream is a virtual global stream and not a specific stream. An error will be returned in this case.

When memory is associated with a single stream, the Unified Memory system will allow CPU access to this memory region so long as all operations in {\ttfamily h\+Stream} have completed, regardless of whether other streams are active. In effect, this constrains exclusive ownership of the managed memory region by an active GPU to per-\/stream activity instead of whole-\/\+GPU activity.

Accessing memory on the device from streams that are not associated with it will produce undefined results. No error checking is performed by the Unified Memory system to ensure that kernels launched into other streams do not access this region.

It is a program\textquotesingle{}s responsibility to order calls to \doxylink{group___c_u_d_a___s_t_r_e_a_m_ga52c3810ba9616e2e1dcac7247491d829}{cu\+Stream\+Attach\+Mem\+Async} via events, synchronization or other means to ensure legal access to memory at all times. Data visibility and coherency will be changed appropriately for all kernels which follow a stream-\/association change.

If {\ttfamily h\+Stream} is destroyed while data is associated with it, the association is removed and the association reverts to the default visibility of the allocation as specified at \doxylink{group___c_u_d_a___m_e_m_gab3c8db1c575451d53313b05b1c6db7f8}{cu\+Mem\+Alloc\+Managed}. For {\bfseries{managed}} variables, the default association is always \doxylink{group___c_u_d_a___t_y_p_e_s_ggae58977b6b2ea67a3fb0e11949302308ea0b42aae6a29b41b734d4c0dea6c33313}{CU\+\_\+\+MEM\+\_\+\+ATTACH\+\_\+\+GLOBAL}. Note that destroying a stream is an asynchronous operation, and as a result, the change to default association won\textquotesingle{}t happen until all work in the stream has completed.

Begins graph capture on a stream. 

Begin graph capture on {\ttfamily h\+Stream}. When a stream is in capture mode, all operations pushed into the stream will not be executed, but will instead be captured into a graph, which will be returned via \doxylink{group___c_u_d_a___s_t_r_e_a_m_ga3cf7bb74b5cb1240199d203d850a0d0b}{cu\+Stream\+End\+Capture}. Returns a stream\textquotesingle{}s capture status. 

Return the capture status of {\ttfamily h\+Stream} via {\ttfamily capture\+Status}. After a successful call, {\ttfamily \texorpdfstring{$\ast$}{*}capture\+Status} will contain one of the following\+:
\begin{DoxyItemize}
\item \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{CU\+\_\+\+STREAM\+\_\+\+CAPTURE\+\_\+\+STATUS\+\_\+\+NONE\+:} The stream is not capturing.
\item \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{CU\+\_\+\+STREAM\+\_\+\+CAPTURE\+\_\+\+STATUS\+\_\+\+ACTIVE\+:} The stream is capturing.

\end{DoxyItemize}

Note that, if this is called on \doxylink{group___c_u_d_a___t_y_p_e_s_gaa53e8210837f039dd6434a3a4c3324aa}{CU\+\_\+\+STREAM\+\_\+\+LEGACY} (the "{}null stream"{}) while a blocking stream in the same context is capturing, it will return \doxylink{group___c_u_d_a___t_y_p_e_s_gga0cdead942fd5028d157641eef6bdeeaaa960a55453736ec87ca941f9bc2d80abe}{CUDA\+\_\+\+ERROR\+\_\+\+STREAM\+\_\+\+CAPTURE\+\_\+\+IMPLICIT} and {\ttfamily \texorpdfstring{$\ast$}{*}capture\+Status} is unspecified after the call. The blocking stream capture is not invalidated.

When a blocking stream is capturing, the legacy stream is in an unusable state until the blocking stream capture is terminated. The legacy stream is not supported for stream capture, but attempted use would have an implicit dependency on the capturing stream(s).

Update the set of dependencies in a capturing stream (11.\+3+) 

Modifies the dependency set of a capturing stream. The dependency set is the set of nodes that the next captured node in the stream will depend on.

Valid flags are \doxylink{group___c_u_d_a___t_y_p_e_s_ggafe81581a4ea83b161fc88e6aeea339cdabab808cd5e4e683f7000cb109973604e}{CU\+\_\+\+STREAM\+\_\+\+ADD\+\_\+\+CAPTURE\+\_\+\+DEPENDENCIES} and \doxylink{group___c_u_d_a___t_y_p_e_s_ggafe81581a4ea83b161fc88e6aeea339cdae3ada3eef9666e592a2d4c3301d08fca}{CU\+\_\+\+STREAM\+\_\+\+SET\+\_\+\+CAPTURE\+\_\+\+DEPENDENCIES}. These control whether the set passed to the API is added to the existing set or replaces it. A flags value of 0 defaults to \doxylink{group___c_u_d_a___t_y_p_e_s_ggafe81581a4ea83b161fc88e6aeea339cdabab808cd5e4e683f7000cb109973604e}{CU\+\_\+\+STREAM\+\_\+\+ADD\+\_\+\+CAPTURE\+\_\+\+DEPENDENCIES}.

Nodes that are removed from the dependency set via this API do not result in \doxylink{group___c_u_d_a___t_y_p_e_s_gga0cdead942fd5028d157641eef6bdeeaaa426e5dd5af746f6ee25aeb0f9fd32402}{CUDA\+\_\+\+ERROR\+\_\+\+STREAM\+\_\+\+CAPTURE\+\_\+\+UNJOINED} if they are unreachable from the stream at \doxylink{group___c_u_d_a___s_t_r_e_a_m_ga3cf7bb74b5cb1240199d203d850a0d0b}{cu\+Stream\+End\+Capture}.

During stream capture (see cu\+Stream\+Begin\+Capture), some actions, such as a call to \doxylink{group___c_u_d_a_r_t___m_e_m_o_r_y_ga2ec57cea7c57e8e3c4beca43dd327134}{cuda\+Malloc}, may be unsafe. In the case of \doxylink{group___c_u_d_a_r_t___m_e_m_o_r_y_ga2ec57cea7c57e8e3c4beca43dd327134}{cuda\+Malloc}, the operation is not enqueued asynchronously to a stream, and is not observed by stream capture. Therefore, if the sequence of operations captured via cu\+Stream\+Begin\+Capture depended on the allocation being replayed whenever the graph is launched, the captured graph would be invalid.

Therefore, stream capture places restrictions on API calls that can be made within or concurrently to a cu\+Stream\+Begin\+Capture-\/\doxylink{group___c_u_d_a___s_t_r_e_a_m_ga3cf7bb74b5cb1240199d203d850a0d0b}{cu\+Stream\+End\+Capture} sequence. This behavior can be controlled via this API and flags to cu\+Stream\+Begin\+Capture.

A thread\textquotesingle{}s mode is one of the following\+:
\begin{DoxyItemize}
\item {\ttfamily CU\+\_\+\+STREAM\+\_\+\+CAPTURE\+\_\+\+MODE\+\_\+\+GLOBAL\+:} This is the default mode. If the local thread has an ongoing capture sequence that was not initiated with {\ttfamily CU\+\_\+\+STREAM\+\_\+\+CAPTURE\+\_\+\+MODE\+\_\+\+RELAXED} at {\ttfamily cu\+Stream\+Begin\+Capture}, or if any other thread has a concurrent capture sequence initiated with {\ttfamily CU\+\_\+\+STREAM\+\_\+\+CAPTURE\+\_\+\+MODE\+\_\+\+GLOBAL}, this thread is prohibited from potentially unsafe API calls.
\item {\ttfamily CU\+\_\+\+STREAM\+\_\+\+CAPTURE\+\_\+\+MODE\+\_\+\+THREAD\+\_\+\+LOCAL\+:} If the local thread has an ongoing capture sequence not initiated with {\ttfamily CU\+\_\+\+STREAM\+\_\+\+CAPTURE\+\_\+\+MODE\+\_\+\+RELAXED}, it is prohibited from potentially unsafe API calls. Concurrent capture sequences in other threads are ignored.
\item {\ttfamily CU\+\_\+\+STREAM\+\_\+\+CAPTURE\+\_\+\+MODE\+\_\+\+RELAXED\+:} The local thread is not prohibited from potentially unsafe API calls. Note that the thread is still prohibited from API calls which necessarily conflict with stream capture, for example, attempting \doxylink{group___c_u_d_a___e_v_e_n_t_ga862b4b51fb9fa95371a83c722d10fbc8}{cu\+Event\+Query} on an event that was last recorded inside a capture sequence.
\end{DoxyItemize}
