\doxysubsection{Detailed Description}
\+\_\+\+\_\+\+\_\+\+MANBRIEF\+\_\+\+\_\+\+\_\+ interactions between CUDA Driver API and CUDA Runtime API (\+\_\+\+\_\+\+\_\+\+CURRENT\+\_\+\+FILE\+\_\+\+\_\+\+\_\+) \+\_\+\+\_\+\+\_\+\+ENDMANBRIEF\+\_\+\+\_\+\+\_\+

This section describes the interactions between the CUDA Driver API and the CUDA Runtime API\hypertarget{group___c_u_d_a_r_t___d_r_i_v_e_r_CUDART_CUDA_primary}{}\doxysubsection{\texorpdfstring{Primary Contexts}{Primary Contexts}}\label{group___c_u_d_a_r_t___d_r_i_v_e_r_CUDART_CUDA_primary}
There exists a one to one relationship between CUDA devices in the CUDA Runtime API and \doxylink{group___c_u_d_a___t_y_p_e_s_gaf9f5bd81658f866613785b3a0bb7d7d9}{CUcontext} s in the CUDA Driver API within a process. The specific context which the CUDA Runtime API uses for a device is called the device\textquotesingle{}s primary context. From the perspective of the CUDA Runtime API, a device and its primary context are synonymous.\hypertarget{group___c_u_d_a_r_t___d_r_i_v_e_r_CUDART_CUDA_init}{}\doxysubsection{\texorpdfstring{Initialization and Tear-\/\+Down}{Initialization and Tear-\/\+Down}}\label{group___c_u_d_a_r_t___d_r_i_v_e_r_CUDART_CUDA_init}
CUDA Runtime API calls operate on the CUDA Driver API \doxylink{group___c_u_d_a___t_y_p_e_s_gaf9f5bd81658f866613785b3a0bb7d7d9}{CUcontext} which is current to to the calling host thread. ~\newline


The function \doxylink{group___c_u_d_a_r_t___d_e_v_i_c_e_ga7b63629909800b614427aae92b22d606}{cuda\+Set\+Device()} makes the primary context for the specified device current to the calling thread by calling \doxylink{group___c_u_d_a___c_t_x_gaf99108a1d0cb2e1f4b410a92f64323f8}{cu\+Ctx\+Set\+Current()}.

The CUDA Runtime API will automatically initialize the primary context for a device at the first CUDA Runtime API call which requires an active context. If no \doxylink{group___c_u_d_a___t_y_p_e_s_gaf9f5bd81658f866613785b3a0bb7d7d9}{CUcontext} is current to the calling thread when a CUDA Runtime API call which requires an active context is made, then the primary context for a device will be selected, made current to the calling thread, and initialized.

The context which the CUDA Runtime API initializes will be initialized using the parameters specified by the CUDA Runtime API functions \doxylink{group___c_u_d_a_r_t___d_e_v_i_c_e_gab07b3a85443eb322b583aa2e79570c28}{cuda\+Set\+Device\+Flags()}, \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{cuda\+D3\+D9\+Set\+Direct3\+DDevice()}, \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{cuda\+D3\+D10\+Set\+Direct3\+DDevice()}, \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{cuda\+D3\+D11\+Set\+Direct3\+DDevice()}, \doxylink{group___c_u_d_a_r_t___o_p_e_n_g_l___d_e_p_r_e_c_a_t_e_d_ga45894cfc7e9f2dbe392f4337c4903927}{cuda\+GLSet\+GLDevice()}, and \doxylink{group___c_u_d_a_r_t___v_d_p_a_u_ga0075d09cc621cfe17e05a4d70c534d64}{cuda\+VDPAUSet\+VDPAUDevice()}. Note that these functions will fail with cuda\+Error\+Set\+On\+Active\+Process if they are called when the primary context for the specified device has already been initialized. (or if the current device has already been initialized, in the case of \doxylink{group___c_u_d_a_r_t___d_e_v_i_c_e_gab07b3a85443eb322b583aa2e79570c28}{cuda\+Set\+Device\+Flags()}).

Primary contexts will remain active until they are explicitly deinitialized using \doxylink{group___c_u_d_a_r_t___d_e_v_i_c_e_gadf472c0854e8464c58659e2bde63eee9}{cuda\+Device\+Reset()}. The function \doxylink{group___c_u_d_a_r_t___d_e_v_i_c_e_gadf472c0854e8464c58659e2bde63eee9}{cuda\+Device\+Reset()} will deinitialize the primary context for the calling thread\textquotesingle{}s current device immediately. The context will remain current to all of the threads that it was current to. The next CUDA Runtime API call on any thread which requires an active context will trigger the reinitialization of that device\textquotesingle{}s primary context.

Note that primary contexts are shared resources. It is recommended that the primary context not be reset except just before exit or to recover from an unspecified launch failure.\hypertarget{group___c_u_d_a_r_t___d_r_i_v_e_r_CUDART_CUDA_context}{}\doxysubsection{\texorpdfstring{Context Interoperability}{Context Interoperability}}\label{group___c_u_d_a_r_t___d_r_i_v_e_r_CUDART_CUDA_context}
Note that the use of multiple \doxylink{group___c_u_d_a___t_y_p_e_s_gaf9f5bd81658f866613785b3a0bb7d7d9}{CUcontext} s per device within a single process will substantially degrade performance and is strongly discouraged. Instead, it is highly recommended that the implicit one-\/to-\/one device-\/to-\/context mapping for the process provided by the CUDA Runtime API be used.

If a non-\/primary \doxylink{group___c_u_d_a___t_y_p_e_s_gaf9f5bd81658f866613785b3a0bb7d7d9}{CUcontext} created by the CUDA Driver API is current to a thread then the CUDA Runtime API calls to that thread will operate on that \doxylink{group___c_u_d_a___t_y_p_e_s_gaf9f5bd81658f866613785b3a0bb7d7d9}{CUcontext}, with some exceptions listed below. Interoperability between data types is discussed in the following sections.

The function \doxylink{group___c_u_d_a_r_t___u_n_i_f_i_e_d_gafdc25be0be9b7eed05fa659b80bd82e9}{cuda\+Pointer\+Get\+Attributes()} will return the error cuda\+Error\+Incompatible\+Driver\+Context if the pointer being queried was allocated by a non-\/primary context. The function \doxylink{group___c_u_d_a_r_t___p_e_e_r_ga7443ed162ccf7463a7caf15dc799cc6d}{cuda\+Device\+Enable\+Peer\+Access()} and the rest of the peer access API may not be called when a non-\/primary \doxylink{group___c_u_d_a___t_y_p_e_s_gaf9f5bd81658f866613785b3a0bb7d7d9}{CUcontext} is current. ~\newline
 To use the pointer query and peer access APIs with a context created using the CUDA Driver API, it is necessary that the CUDA Driver API be used to access these features.

All CUDA Runtime API state (e.\+g, global variables\textquotesingle{} addresses and values) travels with its underlying \doxylink{group___c_u_d_a___t_y_p_e_s_gaf9f5bd81658f866613785b3a0bb7d7d9}{CUcontext}. In particular, if a \doxylink{group___c_u_d_a___t_y_p_e_s_gaf9f5bd81658f866613785b3a0bb7d7d9}{CUcontext} is moved from one thread to another then all CUDA Runtime API state will move to that thread as well.
