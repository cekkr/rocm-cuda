Add a callback to a compute stream. 

\begin{DoxyNote}{Note}
This function is slated for eventual deprecation and removal. If you do not require the callback to execute in case of a device error, consider using \doxylink{group___c_u_d_a_r_t___e_x_e_c_u_t_i_o_n_gacdeaa185a76c5cbfd737519261c992a0}{cuda\+Launch\+Host\+Func}. Additionally, this function is not supported with \doxylink{group___c_u_d_a_r_t___s_t_r_e_a_m_ga12bf98fdc63313161005af98225106e5}{cuda\+Stream\+Begin\+Capture} and \doxylink{group___c_u_d_a_r_t___s_t_r_e_a_m_gafd11c141954bc5928667d87a3f4eee29}{cuda\+Stream\+End\+Capture}, unlike \doxylink{group___c_u_d_a_r_t___e_x_e_c_u_t_i_o_n_gacdeaa185a76c5cbfd737519261c992a0}{cuda\+Launch\+Host\+Func}.
\end{DoxyNote}
Adds a callback to be called on the host after all currently enqueued items in the stream have completed. For each cuda\+Stream\+Add\+Callback call, a callback will be executed exactly once. The callback will block later work in the stream until it is finished.

The callback may be passed cuda\+Success or an error code. In the event of a device error, all subsequently executed callbacks will receive an appropriate \doxylink{group___c_u_d_a_r_t___t_y_p_e_s_ga15962a7f6f36e86b4041cd482f69ff10}{cuda\+Error\+\_\+t}.

Callbacks must not make any CUDA API calls. Attempting to use CUDA APIs may result in cuda\+Error\+Not\+Permitted. Callbacks must not perform any synchronization that may depend on outstanding device work or other callbacks that are not mandated to run earlier. Callbacks without a mandated order (in independent streams) execute in undefined order and may be serialized.

For the purposes of Unified Memory, callback execution makes a number of guarantees\+: 
\begin{DoxyItemize}
\item The callback stream is considered idle for the duration of the callback. Thus, for example, a callback may always use memory attached to the callback stream. 
\item The start of execution of a callback has the same effect as synchronizing an event recorded in the same stream immediately prior to the callback. It thus synchronizes streams which have been "{}joined"{} prior to the callback. 
\item Adding device work to any stream does not have the effect of making the stream active until all preceding callbacks have executed. Thus, for example, a callback might use global attached memory even if work has been added to another stream, if it has been properly ordered with an event. 
\item Completion of a callback does not cause a stream to become active except as described above. The callback stream will remain idle if no device work follows the callback, and will remain idle across consecutive callbacks without device work in between. Thus, for example, stream synchronization can be done by signaling from a callback at the end of the stream. 
\end{DoxyItemize}

Attach memory to a stream asynchronously. 

Enqueues an operation in {\ttfamily stream} to specify stream association of {\ttfamily length} bytes of memory starting from {\ttfamily dev\+Ptr}. This function is a stream-\/ordered operation, meaning that it is dependent on, and will only take effect when, previous work in stream has completed. Any previous association is automatically replaced.

{\ttfamily dev\+Ptr} must point to an one of the following types of memories\+:
\begin{DoxyItemize}
\item managed memory declared using the {\bfseries{managed}} keyword or allocated with \doxylink{group___c_u_d_a_r_t___m_e_m_o_r_y_gac85541d7e0fa6bdaeed967fee1c8e780}{cuda\+Malloc\+Managed}.
\item a valid host-\/accessible region of system-\/allocated pageable memory. This type of memory may only be specified if the device associated with the stream reports a non-\/zero value for the device attribute cuda\+Dev\+Attr\+Pageable\+Memory\+Access.
\end{DoxyItemize}

For managed allocations, {\ttfamily length} must be either zero or the entire allocation\textquotesingle{}s size. Both indicate that the entire allocation\textquotesingle{}s stream association is being changed. Currently, it is not possible to change stream association for a portion of a managed allocation.

For pageable allocations, {\ttfamily length} must be non-\/zero.

The stream association is specified using {\ttfamily flags} which must be one of \doxylink{group___c_u_d_a_r_t___t_y_p_e_s_ga4808e47eba73eb94622ec70a9f9b91ff}{cuda\+Mem\+Attach\+Global}, \doxylink{group___c_u_d_a_r_t___t_y_p_e_s_ga4f9a428d18fdd89a99441d0dd27131c0}{cuda\+Mem\+Attach\+Host} or \doxylink{group___c_u_d_a_r_t___t_y_p_e_s_gadc191442d08fc3a9de4cf055edfd2dbe}{cuda\+Mem\+Attach\+Single}. The default value for {\ttfamily flags} is \doxylink{group___c_u_d_a_r_t___t_y_p_e_s_gadc191442d08fc3a9de4cf055edfd2dbe}{cuda\+Mem\+Attach\+Single} If the \doxylink{group___c_u_d_a_r_t___t_y_p_e_s_ga4808e47eba73eb94622ec70a9f9b91ff}{cuda\+Mem\+Attach\+Global} flag is specified, the memory can be accessed by any stream on any device. If the \doxylink{group___c_u_d_a_r_t___t_y_p_e_s_ga4f9a428d18fdd89a99441d0dd27131c0}{cuda\+Mem\+Attach\+Host} flag is specified, the program makes a guarantee that it won\textquotesingle{}t access the memory on the device from any stream on a device that has a zero value for the device attribute cuda\+Dev\+Attr\+Concurrent\+Managed\+Access. If the \doxylink{group___c_u_d_a_r_t___t_y_p_e_s_gadc191442d08fc3a9de4cf055edfd2dbe}{cuda\+Mem\+Attach\+Single} flag is specified and {\ttfamily stream} is associated with a device that has a zero value for the device attribute cuda\+Dev\+Attr\+Concurrent\+Managed\+Access, the program makes a guarantee that it will only access the memory on the device from {\ttfamily stream}. It is illegal to attach singly to the NULL stream, because the NULL stream is a virtual global stream and not a specific stream. An error will be returned in this case.

When memory is associated with a single stream, the Unified Memory system will allow CPU access to this memory region so long as all operations in {\ttfamily stream} have completed, regardless of whether other streams are active. In effect, this constrains exclusive ownership of the managed memory region by an active GPU to per-\/stream activity instead of whole-\/\+GPU activity.

Accessing memory on the device from streams that are not associated with it will produce undefined results. No error checking is performed by the Unified Memory system to ensure that kernels launched into other streams do not access this region.

It is a program\textquotesingle{}s responsibility to order calls to \doxylink{group___c_u_d_a_r_t___s_t_r_e_a_m_gab873a3bc43def9456d72f9561fe79e65}{cuda\+Stream\+Attach\+Mem\+Async} via events, synchronization or other means to ensure legal access to memory at all times. Data visibility and coherency will be changed appropriately for all kernels which follow a stream-\/association change.

If {\ttfamily stream} is destroyed while data is associated with it, the association is removed and the association reverts to the default visibility of the allocation as specified at \doxylink{group___c_u_d_a_r_t___m_e_m_o_r_y_gac85541d7e0fa6bdaeed967fee1c8e780}{cuda\+Malloc\+Managed}. For {\bfseries{managed}} variables, the default association is always \doxylink{group___c_u_d_a_r_t___t_y_p_e_s_ga4808e47eba73eb94622ec70a9f9b91ff}{cuda\+Mem\+Attach\+Global}. Note that destroying a stream is an asynchronous operation, and as a result, the change to default association won\textquotesingle{}t happen until all work in the stream has completed.

Update the set of dependencies in a capturing stream (11.\+3+) 

Modifies the dependency set of a capturing stream. The dependency set is the set of nodes that the next captured node in the stream will depend on.

Valid flags are cuda\+Stream\+Add\+Capture\+Dependencies and \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{cuda\+Stream\+Set\+Capture\+Dependencies}. These control whether the set passed to the API is added to the existing set or replaces it. A flags value of 0 defaults to cuda\+Stream\+Add\+Capture\+Dependencies.

Nodes that are removed from the dependency set via this API do not result in cuda\+Error\+Stream\+Capture\+Unjoined if they are unreachable from the stream at \doxylink{group___c_u_d_a_r_t___s_t_r_e_a_m_gafd11c141954bc5928667d87a3f4eee29}{cuda\+Stream\+End\+Capture}.

Returns cuda\+Error\+Illegal\+State if the stream is not capturing.

This API is new in CUDA 11.\+3. Developers requiring compatibility across minor versions of the CUDA driver to 11.\+0 should not use this API or provide a fallback.

During stream capture (see \doxylink{group___c_u_d_a_r_t___s_t_r_e_a_m_ga12bf98fdc63313161005af98225106e5}{cuda\+Stream\+Begin\+Capture}), some actions, such as a call to \doxylink{group___c_u_d_a_r_t___m_e_m_o_r_y_ga2ec57cea7c57e8e3c4beca43dd327134}{cuda\+Malloc}, may be unsafe. In the case of \doxylink{group___c_u_d_a_r_t___m_e_m_o_r_y_ga2ec57cea7c57e8e3c4beca43dd327134}{cuda\+Malloc}, the operation is not enqueued asynchronously to a stream, and is not observed by stream capture. Therefore, if the sequence of operations captured via \doxylink{group___c_u_d_a_r_t___s_t_r_e_a_m_ga12bf98fdc63313161005af98225106e5}{cuda\+Stream\+Begin\+Capture} depended on the allocation being replayed whenever the graph is launched, the captured graph would be invalid.

Therefore, stream capture places restrictions on API calls that can be made within or concurrently to a \doxylink{group___c_u_d_a_r_t___s_t_r_e_a_m_ga12bf98fdc63313161005af98225106e5}{cuda\+Stream\+Begin\+Capture}-\/\doxylink{group___c_u_d_a_r_t___s_t_r_e_a_m_gafd11c141954bc5928667d87a3f4eee29}{cuda\+Stream\+End\+Capture} sequence. This behavior can be controlled via this API and flags to \doxylink{group___c_u_d_a_r_t___s_t_r_e_a_m_ga12bf98fdc63313161005af98225106e5}{cuda\+Stream\+Begin\+Capture}.

A thread\textquotesingle{}s mode is one of the following\+:
\begin{DoxyItemize}
\item {\ttfamily cuda\+Stream\+Capture\+Mode\+Global\+:} This is the default mode. If the local thread has an ongoing capture sequence that was not initiated with {\ttfamily cuda\+Stream\+Capture\+Mode\+Relaxed} at {\ttfamily cu\+Stream\+Begin\+Capture}, or if any other thread has a concurrent capture sequence initiated with {\ttfamily cuda\+Stream\+Capture\+Mode\+Global}, this thread is prohibited from potentially unsafe API calls.
\item {\ttfamily cuda\+Stream\+Capture\+Mode\+Thread\+Local\+:} If the local thread has an ongoing capture sequence not initiated with {\ttfamily cuda\+Stream\+Capture\+Mode\+Relaxed}, it is prohibited from potentially unsafe API calls. Concurrent capture sequences in other threads are ignored.
\item {\ttfamily cuda\+Stream\+Capture\+Mode\+Relaxed\+:} The local thread is not prohibited from potentially unsafe API calls. Note that the thread is still prohibited from API calls which necessarily conflict with stream capture, for example, attempting \doxylink{group___c_u_d_a_r_t___e_v_e_n_t_gabff85920872d5ed4aaf7b618e392f514}{cuda\+Event\+Query} on an event that was last recorded inside a capture sequence.
\end{DoxyItemize}
