Exit and clean up from CUDA launches. 

\begin{DoxyRefDesc}{Deprecated}
\item[\mbox{\hyperlink{deprecated__deprecated000060}{Deprecated}}]\end{DoxyRefDesc}

Explicitly destroys all cleans up all resources associated with the current device in the current process. Any subsequent API call to this device will reinitialize the device. ~\newline

Returns the preferred cache configuration for the current device. 

\begin{DoxyRefDesc}{Deprecated}
\item[\mbox{\hyperlink{deprecated__deprecated000064}{Deprecated}}]\end{DoxyRefDesc}


Note that this function is deprecated because its name does not reflect its behavior. Its functionality is identical to the non-\/deprecated function \doxylink{group___c_u_d_a_r_t___d_e_v_i_c_e_gaf4b5597c316c70af59499dde077b9345}{cuda\+Device\+Get\+Cache\+Config()}, which should be used instead.

On devices where the L1 cache and shared memory use the same hardware resources, this returns through {\ttfamily p\+Cache\+Config} the preferred cache configuration for the current device. This is only a preference. The runtime will use the requested configuration if possible, but it is free to choose a different configuration if required to execute functions.

This will return a {\ttfamily p\+Cache\+Config} of cuda\+Func\+Cache\+Prefer\+None on devices where the size of the L1 cache and shared memory are fixed.

The supported cache configurations are\+:

\item \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{cuda\+Func\+Cache\+Prefer\+Shared\+:} prefer larger shared memory and smaller L1 cache
\item \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{cuda\+Func\+Cache\+Prefer\+L1\+:} prefer larger L1 cache and smaller shared memory

Sets the preferred cache configuration for the current device. 

\begin{DoxyRefDesc}{Deprecated}
\item[\mbox{\hyperlink{deprecated__deprecated000065}{Deprecated}}]\end{DoxyRefDesc}


Note that this function is deprecated because its name does not reflect its behavior. Its functionality is identical to the non-\/deprecated function \doxylink{group___c_u_d_a_r_t___d_e_v_i_c_e_ga4749a277602c902676e092c0c446ace4}{cuda\+Device\+Set\+Cache\+Config()}, which should be used instead.

On devices where the L1 cache and shared memory use the same hardware resources, this sets through {\ttfamily cache\+Config} the preferred cache configuration for the current device. This is only a preference. The runtime will use the requested configuration if possible, but it is free to choose a different configuration if required to execute the function. Any function preference set via \doxylink{group___c_u_d_a_r_t___e_x_e_c_u_t_i_o_n_ga75ebd65ac35ab892628fbda45bf13372}{cuda\+Func\+Set\+Cache\+Config (C API)} or cuda\+Func\+Set\+Cache\+Config (C++ API) will be preferred over this device-\/wide setting. Setting the device-\/wide cache configuration to cuda\+Func\+Cache\+Prefer\+None will cause subsequent kernel launches to prefer to not change the cache configuration unless required to launch the kernel.

This setting does nothing on devices where the size of the L1 cache and shared memory are fixed.

Launching a kernel with a different preference than the most recent preference setting may insert a device-\/side synchronization point.

The supported cache configurations are\+:
\begin{DoxyItemize}
\item \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{cuda\+Func\+Cache\+Prefer\+None\+:} no preference for shared memory or L1 (default)
\item \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{cuda\+Func\+Cache\+Prefer\+Shared\+:} prefer larger shared memory and smaller L1 cache
\item \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{cuda\+Func\+Cache\+Prefer\+L1\+:} prefer larger L1 cache and smaller shared memory
\end{DoxyItemize}

\begin{DoxyRefDesc}{Deprecated}
\item[\mbox{\hyperlink{deprecated__deprecated000062}{Deprecated}}]\end{DoxyRefDesc}


Note that this function is deprecated because its name does not reflect its behavior. Its functionality is identical to the non-\/deprecated function \doxylink{group___c_u_d_a_r_t___d_e_v_i_c_e_ga2f463a1b29df6299944e0c1004a26f89}{cuda\+Device\+Set\+Limit()}, which should be used instead.

Setting {\ttfamily limit} to {\ttfamily value} is a request by the application to update the current limit maintained by the device. The driver is free to modify the requested value to meet h/w requirements (this could be clamping to minimum or maximum values, rounding up to nearest element size, etc). The application can use \doxylink{group___c_u_d_a_r_t___t_h_r_e_a_d___d_e_p_r_e_c_a_t_e_d_ga78b9a2cfcf2cb0bf84abd5c0c83b8a45}{cuda\+Thread\+Get\+Limit()} to find out exactly what the limit has been set to.

Setting each \doxylink{group___c_u_d_a_r_t___t_y_p_e_s_ga79a610d81adcf0e3fb943f6ff04eb77f}{cuda\+Limit} has its own specific restrictions, so each is discussed here.


\begin{DoxyItemize}
\item cuda\+Limit\+Stack\+Size controls the stack size of each GPU thread.
\item cuda\+Limit\+Printf\+Fifo\+Size controls the size of the shared FIFO used by the \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{printf()} device system call. Setting cuda\+Limit\+Printf\+Fifo\+Size must be performed before launching any kernel that uses the \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{printf()} device system call, otherwise cuda\+Error\+Invalid\+Value will be returned.
\item cuda\+Limit\+Malloc\+Heap\+Size controls the size of the heap used by the \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{malloc()} and \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{free()} device system calls. Setting cuda\+Limit\+Malloc\+Heap\+Size must be performed before launching any kernel that uses the \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{malloc()} or \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{free()} device system calls, otherwise cuda\+Error\+Invalid\+Value will be returned.
\end{DoxyItemize}
