Enables direct access to memory allocations in a peer context. 

If both the current context and {\ttfamily peer\+Context} are on devices which support unified addressing (as may be queried using \doxylink{group___c_u_d_a___t_y_p_e_s_gga3b9f561d2a42733dde99b2cedcaa413aadc11dd6d9f149a7bae32499f2b802c0d}{CU\+\_\+\+DEVICE\+\_\+\+ATTRIBUTE\+\_\+\+UNIFIED\+\_\+\+ADDRESSING}) and same major compute capability, then on success all allocations from {\ttfamily peer\+Context} will immediately be accessible by the current context. See \doxylink{group___c_u_d_a___u_n_i_f_i_e_d}{Unified Addressing} for additional details.

Note that access granted by this call is unidirectional and that in order to access memory from the current context in {\ttfamily peer\+Context}, a separate symmetric call to \doxylink{group___c_u_d_a___p_e_e_r___a_c_c_e_s_s_ga09abdb12526cbcacece926887a76a5e6}{cu\+Ctx\+Enable\+Peer\+Access()} is required.

Note that there are both device-\/wide and system-\/wide limitations per system configuration, as noted in the CUDA Programming Guide under the section "{}\+Peer-\/to-\/\+Peer Memory Access"{}.

Returns \doxylink{group___c_u_d_a___t_y_p_e_s_gga0cdead942fd5028d157641eef6bdeeaaad60abcaa3f2710f961db8c383bb95cae}{CUDA\+\_\+\+ERROR\+\_\+\+PEER\+\_\+\+ACCESS\+\_\+\+UNSUPPORTED} if \doxylink{group___c_u_d_a___p_e_e_r___a_c_c_e_s_s_gadd7611b1357104d94a8d49d16df2e08f}{cu\+Device\+Can\+Access\+Peer()} indicates that the \doxylink{group___c_u_d_a___t_y_p_e_s_ga41ca2a24a242b36ef2ca77330b5fb72a}{CUdevice} of the current context cannot directly access memory from the \doxylink{group___c_u_d_a___t_y_p_e_s_ga41ca2a24a242b36ef2ca77330b5fb72a}{CUdevice} of {\ttfamily peer\+Context}.
