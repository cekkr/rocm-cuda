Find out attributes for a given function. 

This function obtains the attributes of a function specified via {\ttfamily func}. {\ttfamily func} is a device function symbol and must be declared as a {\ttfamily \+\_\+\+\_\+global\+\_\+\+\_\+} function. On devices with configurable shared memory banks, this function will force all subsequent launches of the specified device function to have the given shared memory bank size configuration. On any given launch of the function, the shared memory configuration of the device will be temporarily changed if needed to suit the function\textquotesingle{}s preferred configuration. Launches device functions on multiple devices where thread blocks can cooperate and synchronize as they execute. 

\begin{DoxyRefDesc}{Deprecated}
\item[\mbox{\hyperlink{deprecated__deprecated000066}{Deprecated}}]This function is deprecated as of CUDA 11.\+3.\end{DoxyRefDesc}


Invokes kernels as specified in the {\ttfamily launch\+Params\+List} array where each element of the array specifies all the parameters required to perform a single kernel launch. These kernels can cooperate and synchronize as they execute. The size of the array is specified by {\ttfamily num\+Devices}.

No two kernels can be launched on the same device. All the devices targeted by this multi-\/device launch must be identical. All devices must have a non-\/zero value for the device attribute cuda\+Dev\+Attr\+Cooperative\+Multi\+Device\+Launch.

The same kernel must be launched on all devices. Note that any {\bfseries{device}} or {\bfseries{constant}} variables are independently instantiated on every device. It is the application\textquotesingle{}s responsiblity to ensure these variables are initialized and used appropriately.

The size of the grids as specified in blocks, the size of the blocks themselves and the amount of shared memory used by each thread block must also match across all launched kernels.

The streams used to launch these kernels must have been created via either \doxylink{group___c_u_d_a_r_t___s_t_r_e_a_m_ga11a80128d72f5ecf20072976122c7cd4}{cuda\+Stream\+Create} or \doxylink{group___c_u_d_a_r_t___s_t_r_e_a_m_gaa028ceaec5dc941c48c49678915126bc}{cuda\+Stream\+Create\+With\+Priority} or \doxylink{group___c_u_d_a_r_t___s_t_r_e_a_m_gaa028ceaec5dc941c48c49678915126bc}{cuda\+Stream\+Create\+With\+Priority}. The NULL stream or \doxylink{group___c_u_d_a_r_t___t_y_p_e_s_ga4e62d09dde16ba457b0a97f3a5262246}{cuda\+Stream\+Legacy} or \doxylink{group___c_u_d_a_r_t___t_y_p_e_s_ga7b7129befd6f52708309acafd1c46197}{cuda\+Stream\+Per\+Thread} cannot be used.

The total number of blocks launched per kernel cannot exceed the maximum number of blocks per multiprocessor as returned by \doxylink{group___c_u_d_a_r_t___o_c_c_u_p_a_n_c_y_gae162c2e0a6864271d20cc60bb107abc1}{cuda\+Occupancy\+Max\+Active\+Blocks\+Per\+Multiprocessor} (or \doxylink{group___c_u_d_a_r_t___o_c_c_u_p_a_n_c_y_gafd44b9bd208a9d8bb3fefb9a7a65d425}{cuda\+Occupancy\+Max\+Active\+Blocks\+Per\+Multiprocessor\+With\+Flags}) times the number of multiprocessors as specified by the device attribute cuda\+Dev\+Attr\+Multi\+Processor\+Count. Since the total number of blocks launched per device has to match across all devices, the maximum number of blocks that can be launched per device will be limited by the device with the least number of multiprocessors.

The kernel cannot make use of CUDA dynamic parallelism.

Enqueues a host function call in a stream. 

Enqueues a host function to run in a stream. The function will be called after currently enqueued work and will block work added after it.

The host function must not make any CUDA API calls. Attempting to use a CUDA API may result in cuda\+Error\+Not\+Permitted, but this is not required. The host function must not perform any synchronization that may depend on outstanding CUDA work not mandated to run earlier. Host functions without a mandated order (such as in independent streams) execute in undefined order and may be serialized.

For the purposes of Unified Memory, execution makes a number of guarantees\+: 
\begin{DoxyItemize}
\item The stream is considered idle for the duration of the function\textquotesingle{}s execution. Thus, for example, the function may always use memory attached to the stream it was enqueued in. 
\item The start of execution of the function has the same effect as synchronizing an event recorded in the same stream immediately prior to the function. It thus synchronizes streams which have been "{}joined"{} prior to the function. 
\item Adding device work to any stream does not have the effect of making the stream active until all preceding host functions and stream callbacks have executed. Thus, for example, a function might use global attached memory even if work has been added to another stream, if the work has been ordered behind the function call with an event. 
\item Completion of the function does not cause a stream to become active except as described above. The stream will remain idle if no device work follows the function, and will remain idle across consecutive host functions or stream callbacks without device work in between. Thus, for example, stream synchronization can be done by signaling from a host function at the end of the stream. 
\end{DoxyItemize}

Note that, in constrast to \doxylink{group___c_u_d_a___s_t_r_e_a_m_ga120776d5935ae856369b231dc6dedcbf}{cu\+Stream\+Add\+Callback}, the function will not be called in the event of an error in the CUDA context.

Launches a device function. 

The function invokes kernel {\ttfamily func} on {\ttfamily grid\+Dim} ({\ttfamily grid\+Dim.\+x} {$\times$} {\ttfamily grid\+Dim.\+y} {$\times$} {\ttfamily grid\+Dim.\+z}) grid of blocks. Each block contains {\ttfamily block\+Dim} ({\ttfamily block\+Dim.\+x} {$\times$} {\ttfamily block\+Dim.\+y} {$\times$} {\ttfamily block\+Dim.\+z}) threads.

If the kernel has N parameters the {\ttfamily args} should point to array of N pointers. Each pointer, from {\ttfamily args\mbox{[}0\mbox{]}} to {\ttfamily args\mbox{[}N -\/ 1\mbox{]}}, point to the region of memory from which the actual parameter will be copied.

For templated functions, pass the function symbol as follows\+: func\+\_\+name$<$template\+\_\+arg\+\_\+0,...,template\+\_\+arg\+\_\+\+N$>$

{\ttfamily shared\+Mem} sets the amount of dynamic shared memory that will be available to each thread block.

{\ttfamily stream} specifies a stream the invocation is associated to.
