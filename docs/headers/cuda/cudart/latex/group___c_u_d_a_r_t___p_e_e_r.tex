Returns in {\ttfamily \texorpdfstring{$\ast$}{*}can\+Access\+Peer} a value of 1 if device {\ttfamily device} is capable of directly accessing memory from {\ttfamily peer\+Device} and 0 otherwise. Enables direct access to memory allocations on a peer device. 

On success, all allocations from {\ttfamily peer\+Device} will immediately be accessible by the current device. They will remain accessible until access is explicitly disabled using \doxylink{group___c_u_d_a_r_t___p_e_e_r_gad1d17e928a31f1cb1691a42de480c2d3}{cuda\+Device\+Disable\+Peer\+Access()} or either device is reset using \doxylink{group___c_u_d_a_r_t___d_e_v_i_c_e_gadf472c0854e8464c58659e2bde63eee9}{cuda\+Device\+Reset()}.

Note that access granted by this call is unidirectional and that in order to access memory on the current device from {\ttfamily peer\+Device}, a separate symmetric call to \doxylink{group___c_u_d_a_r_t___p_e_e_r_ga7443ed162ccf7463a7caf15dc799cc6d}{cuda\+Device\+Enable\+Peer\+Access()} is required.

Note that there are both device-\/wide and system-\/wide limitations per system configuration, as noted in the CUDA Programming Guide under the section "{}\+Peer-\/to-\/\+Peer Memory Access"{}.

Returns cuda\+Error\+Invalid\+Device if \doxylink{group___c_u_d_a_r_t___p_e_e_r_ga46cf1454e8daa73d9bc17aa5a1589a38}{cuda\+Device\+Can\+Access\+Peer()} indicates that the current device cannot directly access memory from {\ttfamily peer\+Device}.

Returns cuda\+Error\+Peer\+Access\+Already\+Enabled if direct access of {\ttfamily peer\+Device} from the current device has already been enabled.

Returns cuda\+Error\+Invalid\+Value if {\ttfamily flags} is not 0.
