Mapping two buffers whose ranges overlap may or may not result in the same virtual address being returned for the overlapped portion. Otherwise writes made via one address are not guaranteed to be visible via the other address, even if they\textquotesingle{}re issued by the same thread. It is recommended that applications map the combined range instead of mapping separate buffers and then apply the appropriate offsets to the returned pointer to derive the individual buffers.

By default, waiting on such an external semaphore object causes appropriate memory synchronization operations to be performed over all external memory objects that are imported as \doxylink{group___c_u_d_a___t_y_p_e_s_ggafe2bee7f51d487e11f460752088812a8aadf93206205d155aed8228f4a118d6ee}{CU\+\_\+\+EXTERNAL\+\_\+\+MEMORY\+\_\+\+HANDLE\+\_\+\+TYPE\+\_\+\+NVSCIBUF}. This ensures that any subsequent accesses made by other importers of the same set of Nv\+Sci\+Buf memory object(s) are coherent. These operations can be skipped by specifying the flag \doxylink{group___c_u_d_a___t_y_p_e_s_gaf07a1d15f2696b915c068c892e6f1a35}{CUDA\+\_\+\+EXTERNAL\+\_\+\+SEMAPHORE\+\_\+\+WAIT\+\_\+\+SKIP\+\_\+\+NVSCIBUF\+\_\+\+MEMSYNC}, which can be used as a performance optimization when data coherency is not required. But specifying this flag in scenarios where data coherency is required results in undefined behavior. Also, for semaphore object of the type \doxylink{group___c_u_d_a___t_y_p_e_s_ggad8b911cfa4a4cf57761be3f87e91d3efa908455daa7bbeb91a83930f977c0f1c1}{CU\+\_\+\+EXTERNAL\+\_\+\+SEMAPHORE\+\_\+\+HANDLE\+\_\+\+TYPE\+\_\+\+NVSCISYNC}, if the Nv\+Sci\+Sync\+Attr\+List used to create the Nv\+Sci\+Sync\+Obj had not set the flags in \doxylink{group___c_u_d_a___d_e_v_i_c_e_ga1bfd645a691e1cd33873c94605d2089a}{cu\+Device\+Get\+Nv\+Sci\+Sync\+Attributes} to CUDA\+\_\+\+NVSCISYNC\+\_\+\+ATTR\+\_\+\+WAIT, this API will return CUDA\+\_\+\+ERROR\+\_\+\+NOT\+\_\+\+SUPPORTED.

If the semaphore object is any one of the following types\+: \doxylink{group___c_u_d_a___t_y_p_e_s_ggad8b911cfa4a4cf57761be3f87e91d3efaa93c602f3eb2b39ab95b0dfeb72f8eba}{CU\+\_\+\+EXTERNAL\+\_\+\+SEMAPHORE\+\_\+\+HANDLE\+\_\+\+TYPE\+\_\+\+D3\+D11\+\_\+\+KEYED\+\_\+\+MUTEX}, \doxylink{group___c_u_d_a___t_y_p_e_s_ggad8b911cfa4a4cf57761be3f87e91d3efac97816c259dde37432b63177377473da}{CU\+\_\+\+EXTERNAL\+\_\+\+SEMAPHORE\+\_\+\+HANDLE\+\_\+\+TYPE\+\_\+\+D3\+D11\+\_\+\+KEYED\+\_\+\+MUTEX\+\_\+\+KMT} then the keyed mutex will be acquired when it is released with the key specified in \doxylink{struct_c_u_d_a___e_x_t_e_r_n_a_l___s_e_m_a_p_h_o_r_e___w_a_i_t___p_a_r_a_m_s__st_ae08747d65535a068feb8c37eb55e7805}{CUDA\+\_\+\+EXTERNAL\+\_\+\+SEMAPHORE\+\_\+\+WAIT\+\_\+\+PARAMS\+::params\+::keyedmutex\+::key} or until the timeout specified by \doxylink{struct_c_u_d_a___e_x_t_e_r_n_a_l___s_e_m_a_p_h_o_r_e___w_a_i_t___p_a_r_a_m_s__st_a892e6e48477a8a08b73e345ab85d52c6}{CUDA\+\_\+\+EXTERNAL\+\_\+\+SEMAPHORE\+\_\+\+WAIT\+\_\+\+PARAMS\+::params\+::keyedmutex\+::timeout\+Ms} has lapsed. The timeout interval can either be a finite value specified in milliseconds or an infinite value. In case an infinite value is specified the timeout never elapses. The windows INFINITE macro must be used to specify infinite timeout.
