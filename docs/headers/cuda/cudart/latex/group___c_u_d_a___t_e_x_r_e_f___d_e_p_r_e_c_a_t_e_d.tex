If the device memory pointer was returned from cu\+Mem\+Alloc(), the offset is guaranteed to be 0 and NULL may be passed as the {\ttfamily Byte\+Offset} parameter.

The total number of elements (or texels) in the linear address range cannot exceed \doxylink{group___c_u_d_a___t_y_p_e_s_gga3b9f561d2a42733dde99b2cedcaa413aa335584a4bc5128e2a5ae9a4417f5b758}{CU\+\_\+\+DEVICE\+\_\+\+ATTRIBUTE\+\_\+\+MAXIMUM\+\_\+\+TEXTURE1\+D\+\_\+\+LINEAR\+\_\+\+WIDTH}. \item \doxylink{group___c_u_d_a___t_y_p_e_s_ga6d2387c1b5dd5bc98f5b4c51cefdf41e}{CU\+\_\+\+TRSF\+\_\+\+READ\+\_\+\+AS\+\_\+\+INTEGER}, which suppresses the default behavior of having the texture promote integer data to floating point data in the range \mbox{[}0, 1\mbox{]}. Note that texture with 32-\/bit integer format would not be promoted, regardless of whether or not this flag is specified;
\item \doxylink{group___c_u_d_a___t_y_p_e_s_ga7a19eb49fd506ecded6e8f314298d486}{CU\+\_\+\+TRSF\+\_\+\+NORMALIZED\+\_\+\+COORDINATES}, which suppresses the default behavior of having the texture coordinates range from \mbox{[}0, Dim) where Dim is the width or height of the CUDA array. Instead, the texture coordinates \mbox{[}0, 1.\+0) reference the entire breadth of the array dimension;
\item \doxylink{group___c_u_d_a___t_y_p_e_s_ga9512d7fa0ed8a2da30ef6f4ccc61fa4f}{CU\+\_\+\+TRSF\+\_\+\+DISABLE\+\_\+\+TRILINEAR\+\_\+\+OPTIMIZATION}, which disables any trilinear filtering optimizations. Trilinear optimizations improve texture filtering performance by allowing bilinear filtering on textures in scenarios where it can closely approximate the expected results.
\end{DoxyItemize}
