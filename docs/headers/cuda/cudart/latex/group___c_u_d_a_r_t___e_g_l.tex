Additional Mapping/\+Unmapping is not required for the registered resource and \doxylink{group___c_u_d_a_r_t___e_g_l_ga5c511b01f2e3369d563e4f7c61cc57e7}{cuda\+Graphics\+Resource\+Get\+Mapped\+Egl\+Frame} can be directly called on the {\ttfamily p\+Cuda\+Resource}.

The application will be responsible for synchronizing access to shared objects. The application must ensure that any pending operation which access the objects have completed before passing control to CUDA. This may be accomplished by issuing and waiting for gl\+Finish command on all GLcontexts (for Open\+GL and likewise for other APIs). The application will be also responsible for ensuring that any pending operation on the registered CUDA resource has completed prior to executing subsequent commands in other APIs accesing the same memory objects. This can be accomplished by calling cu\+Ctx\+Synchronize or cu\+Event\+Synchronize (preferably).

The surface\textquotesingle{}s intended usage is specified using {\ttfamily flags}, as follows\+:


\begin{DoxyItemize}
\item \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{cuda\+Graphics\+Register\+Flags\+None\+:} Specifies no hints about how this resource will be used. It is therefore assumed that this resource will be read from and written to by CUDA. This is the default value.
\item \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{cuda\+Graphics\+Register\+Flags\+Read\+Only\+:} Specifies that CUDA will not write to this resource.
\item \doxylink{group___c_u_d_a_r_t___t_e_x_t_u_r_e___o_b_j_e_c_t_ga91840d654e4a4ffb86e9b2fd793e9196}{cuda\+Graphics\+Register\+Flags\+Write\+Discard\+:} Specifies that CUDA will not read from this resource and will write over the entire contents of the resource, so none of the data previously stored in the resource will be preserved.
\end{DoxyItemize}
